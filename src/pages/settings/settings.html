How it works

The app will use an API endpoint to interact with the Meraki Dasbhoard. If using Meraki directly, use https://dashboard.meraki.com/api/vo and enter your API key. For any calls that require adds, changes or deletes, it is best to use the dashoard link associated with your organization. For instance, the URL will include a number instead of dashboard: http://n123.meraki.com/api/v0. This simply avoids issues enountered when Meraki redirects the client to the correct "shard", which is represented by the number.

The other option is to use your own API endpoint or proxy service. This may be preferred for security or performance reasons. In that case, an API key is only required if that service requires it.
